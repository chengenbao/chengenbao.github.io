## 1. TorchForge + Weaver：大规模 RL 后训练新范式

**来源：** [PyTorch Blog](https://pytorch.org/blog/supercharging-llms-scalable-rl-with-torchforge-and-weaver/) | **日期：** 2026-01-09

Meta PyTorch 团队开源 **torchforge**，在 512-GPU 集群上运行 GRPO，单步训练时间从 6 分钟压缩至 1.5 分钟（4× 加速），GPU 利用率 >65%。核心是 Forge（PyTorch 原生 RL 原语）+ Weaver（400M 参数弱验证器聚合，支持每查询 100+ 候选评估）+ TorchStore（RDMA 权重同步）。Weaver 在 MATH-500 上 Qwen3-8B 达 77.5%，32B 达 82.9%，全程无人工标注。

## 2. TorchTitan + AMD MI325X：1K GPU MoE 预训练近线性扩展

**来源：** [PyTorch Blog](https://pytorch.org/blog/efficient-moe-pre-training-at-scale-with-torchtitan/) | **日期：** 2025-12-01

AMD+Meta 合作在 1024 块 MI325X GPU 上完成 DeepSeek-V3-671B 预训练：扩展效率 96%（128→1024 GPU），端到端吞吐 2.77×。关键手段：Primus-Turbo FP8 grouped GEMM、DeepEP（EP=32 时通信占比压至 18%）、VPP 流水线调度。Llama 4-Scout 实现 100-102% 线性扩展（32→512 GPU）。全栈开源。

## 3. HuggingFace Transformers v5：MoE 一等公民支持

**来源：** [Hugging Face Blog](https://huggingface.co/blog/moe-transformers) | **日期：** 2026-02-26

三大改进：① WeightConverter 异步加载（Qwen 110B 加载时间 66s→10s）；② 插件式 Expert Backend（eager/batched_mm/grouped_mm）；③ Expert Parallelism（`enable_expert_parallel=True`）。联合 Unsloth 实现 MoE 训练速度 ~12×，VRAM 减少 >35%。

## 4. AI Agent 自动生成生产级 CUDA Kernel

**来源：** [Hugging Face Blog](https://huggingface.co/blog/custom-cuda-kernels-agent-skills) | **日期：** 2026-02-13

HF 发布 CUDA Kernel Agent Skill，向 Claude/Codex 注入 GPU 架构知识，自动生成可部署内核。实测：LTX-Video RMSNorm 平均 1.88× 提速，Qwen3-8B 在 8192 token 时 2.47×。一键安装：`kernels skills add cuda-kernels --claude`，生成项目可直接推送 HF Kernel Hub。
