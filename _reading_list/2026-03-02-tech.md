## 1. 用 RL 加速 LLM 后训练：torchforge + Weaver
**来源：** [PyTorch Blog](https://pytorch.org/blog/supercharging-llms-scalable-rl-with-torchforge-and-weaver/)  
**领域：** 强化学习 / LLM 后训练  
**合作方：** Stanford、Meta、CoreWeave

大规模强化学习用于 LLM 后训练一直被认为极具挑战性。PyTorch 联合 Stanford、Meta、CoreWeave 推出 **torchforge** 框架与 **Weaver** 系统，通过高效的 RL 训练基础设施解决了扩展性瓶颈。展示了在千卡 GPU 集群上对大语言模型进行后训练强化学习的完整工程方案，是 RLHF/RLVR 工程化落地的重要里程碑。

**关键词：** `torchforge` `Weaver` `RLHF` `后训练` `scalable RL`

## 2. MoE 架构深度解析：Transformers 中的混合专家
**来源：** [Hugging Face Blog](https://huggingface.co/blog/moe-transformers)（2026-02-26）  
**领域：** 模型架构 / 高效推理

Hugging Face 发布 MoE（混合专家）完整技术解析，揭示为何稀疏 MoE 成为当代大模型首选架构：
- 核心思路：用稀疏激活代替全量计算——GPT-OSS-20B 有 21B 总参数，但每 token 仅使用约 3.6B 活跃参数
- DeepSeek-V3、Llama 4-Scout 均采用 MoE 架构
- 详解 HuggingFace Transformers 库中的 MoE 高效推理实现

**关键词：** `MoE` `稀疏激活` `专家路由` `DeepSeek-V3` `Llama 4`

## 3. Teaching AI to Read a Map：MapTrace 空间推理新基准
**来源：** [Google Research Blog](https://research.google/blog/teaching-ai-to-read-a-map/)（2026-02-17）  
**领域：** 多模态 / 空间推理 / 合成数据

Google Research 提出 MapTrace 任务与数据集，专门解决 MLLM 在地图路径规划上的空间推理失败问题：
- 2M 问答对，使用 Gemini 2.5 Pro + Imagen-4 合成生成

- 论文：[MapTrace: Scalable Data Generation for Route Tracing on Maps](https://arxiv.org/abs/2512.19609)

**关键词：** `MapTrace` `MLLM` `空间推理` `合成数据` `Gemini`

## 4. Sequential Attention：让 AI 模型更精简更快速
**来源：** [Google Research Blog](https://research.google/blog/sequential-attention-making-ai-models-leaner-and-faster-without-sacrificing-accuracy/)（2026-02-04）

## 5. CORPGEN：AI Agent 多任务并行的新范式
**来源：** [Microsoft Research Blog](https://www.microsoft.com/en-us/research/blog/corpgen-advances-ai-agents-for-real-work/)（2026-02-26）  
**领域：** AI Agent / 企业应用

微软研究院 CORPGEN 框架：
- 引入 Multi-Horizon Task Environments (MHTEs)：5 小时内同时管理多个需 10-30 步骤的任务
- 主流 Agent 多任务完成率从 16.7% 暴跌至 8.7%
- CORPGEN 通过分层规划、记忆隔离和经验学习，完成率最高提升 **3.5 倍**

**关键词：** `AI Agent` `多任务` `MHTE` `企业AI`

## 6. Amazon Nova 强化微调：通过反馈学习
**来源：** [AWS Machine Learning Blog](https://aws.amazon.com/blogs/machine-learning/reinforcement-fine-tuning-for-amazon-nova-teaching-ai-through-feedback/)  
**领域：** 强化学习 / 模型微调

AWS 将强化微调（RFT）引入 Amazon Nova，主打"通过评估学习而非模仿学习"，只需定义正确答案标准而不需要完整推理路径标注数据。适用于代码生成、金融推理等多路径任务。

**关键词：** `RFT` `强化微调` `Amazon Nova` `Bedrock`
